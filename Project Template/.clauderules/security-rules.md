# Security Rules — SynergyAI
# Layer 3: .clauderules/ — Loaded when auth/security is relevant

## Auth Layers

### 1. External Service ↔ Server (Secret API Token)
- Each customer has a Secret API Token generated by the server.
- Server verifies: Authorization: Bearer <SECRET_TOKEN> on every request.
- Token hashed in DB (bcrypt). Never stored in plaintext.

### 2. End User ↔ Server (JWT Token)
- Stateless sessions — JWT signed with server's Secret Key.
- Server validates: signature, expiry, session state.
- JWT is single-use: invalidated once consumed.

### 3. SaaS Dashboard (Supabase Auth)
- Supabase Auth for everything. No custom auth.
- OAuth: Google as primary provider + email/password.
- Middleware verifies session on protected routes.
- Use getUser() (server-side, secure) over getSession() (client-side, manipulable).

## Agent Data Isolation
- Each company's embeddings namespaced by company_id in pgvector queries.
- RLS on every table filters by company_id (derived from JWT).
- Agent context never includes another company's knowledge base.
- Red-line configuration only visible to owning company.
- Conversation logs only accessible to participating companies + platform admin.
- Trust score breakdown (components) private, only aggregate score shared.

## Blockchain & Wallet Security
- Custodial wallets: platform manages keys, companies never see private keys.
- Platform wallet key stored in HSM (production) or encrypted secret (dev).
- Multi-sig (2-of-3) required for treasury operations.
- Smart contracts audited before mainnet deployment.
- All on-chain transactions logged with tx_hash in audit table.
- SYNRG token amounts never exposed to UI — show "credits" only.

## Input Validation
- Zod on EVERY input: forms, API bodies, URL params, webhook payloads (TypeScript).
- Go: validation with struct tags or validator library.
- Max length on all text fields.

## File Upload Security (if applicable)
- Only accept files if JWT is valid and not expired.
- File size limit: configurable per feature.
- MIME type validation: whitelist only expected types.
- Save with UUID name (not predictable).
- Never serve files directly from a public URL — use signed URLs.

## Rate Limiting
- Public APIs: 30 req/min per IP.
- Authenticated APIs: 100 req/min per user.
- AI conversation endpoints: 10 req/min per company.
- Matching endpoints: 5 req/min per company.
- Settlement endpoints: 3 req/min per company.
- Implement in-memory initially, then Redis-backed.

## GDPR & Privacy
- User data: configurable retention policy.
- Auto-delete: cron job that removes expired data.
- AI calls: not used for model training (opt-out from providers).
- Company deletion cascades to: agents, embeddings, conversations, matches, wallet, credit_transactions.
- Hard delete within 30 days of request.
- On-chain data (hashes only) cannot be deleted — no PII stored on-chain.

## Environment
- Use @t3-oss/env-nextjs for env var validation at build time (dashboard).
- Server secrets never in NEXT_PUBLIC_*.
- .claude/settings.local.json in .gitignore.
- .env.local in .gitignore.
- Go: env vars validated at server startup. Panic if missing critical vars.
